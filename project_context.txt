=== PROJECT OVERVIEW ===
Purpose: AFL match prediction for family tipping competition (learning exercise)
- Using time-based series ML techniques for real-time outcome prediction
- Transparent with family (they can see code + predictions)
- Won last 3 years in a row
- Focus: discovering non-obvious statistical features that predict wins

Approach:
- Retrieve data from afltables website
- Parse into local storage/database
- Sequential training/retraining to optimize models
- Feature importance analysis to find interesting patterns
  (e.g., "teams with X marks and Y tackles are more likely to win by Z%")

Current Implementation:
- Data pipeline: crawl/fetch scripts from afltables
- Database: SQLite with migration support
- Models: XGBoost with time-based train/test splits
- Example: train on 1990-1994, test on 1995
- Hyperparameter search and optimization
- Ensemble methods and stacking
- Feature importance and misprediction analysis

**Root Cause Analysis - Missing Data:**
Models lack contextual information humans use:
1. **Injury news** - Key players out (stars, ruckmen, forwards)
2. **Team selection** - Debuts, returns, omissions (announced Thu/Fri)
3. **Betting odds** - Incorporate ALL public info including injuries
4. **Fixture factors:**
   - Days rest (5-day breaks vs 8+ days)
   - Travel distance (Perth to Melbourne = significant fatigue)
   - Interstate vs home state games
5. **Ladder position** - Finals pressure, dead rubbers
6. **Weather** - Rain affects scoring, wind affects accuracy
7. **Match importance** - Rivalry games, elimination finals
8. **Coaching changes** - New coaches, stability
